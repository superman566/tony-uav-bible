# 深度学习基础

## 什么是人工智能（AI）

**一句话理解**：人工智能 = 让机器表现出"像人一样聪明"的行为，是一个最大的概念范畴，怎么实现都算。

只要一个程序能完成原本需要人类智能才能做的事——下棋、认脸、听懂话、开车——不管它内部是怎么实现的，都可以叫人工智能。早期很多 AI 甚至完全不需要"学习"，纯靠人写规则：

**例子**：一个简单的下棋程序，人可以直接写死规则——"如果对方吃我的车，我就把它的皇后将死"。这种靠 if-else 规则堆出来的程序，也是人工智能的一种，只是比较"笨"，规则之外的情况就不会处理了。

## 什么是机器学习（ML）

**一句话理解**：机器学习 = 实现人工智能的其中一种方法，让机器自己从数据里"总结"规律，而不是人把规则写死。

**例子**：判断一封邮件是不是垃圾邮件。

- 传统 AI 做法：人自己想规则——"包含‘中奖’‘免费’‘点击领取’就是垃圾邮件"，写成 if-else。
- 机器学习做法：给模型看 10 万封已经标好"垃圾/正常"的邮件，模型自己统计出"哪些词、哪些特征更容易出现在垃圾邮件里"，总结出一套规则。

机器学习的核心变化是：**规则不再是人写的，而是从数据里"学"出来的**。常见的机器学习方法有决策树、SVM（支持向量机）、线性回归等，深度学习也是机器学习的一种。

## 监督学习、无监督学习、半监督学习

机器学习"从数据里学规律"，但根据数据**有没有标准答案**、有多少，又能分成三种方式。先解释一下"标准答案"：

**什么是标签（Label）**：给一条数据配上的"正确答案"。比如一封邮件配上"垃圾/正常"，一张图片配上"猫"这个类别，这个"正确答案"就叫标签。

### 监督学习（Supervised Learning）

**一句话理解**：给模型看的每条数据都带着标准答案，模型照着答案学，学的是"输入 → 输出"的对应关系。

**例子**：前面讲线性回归时用的"面积、房龄 → 成交价"就是监督学习——每条训练数据里，价格（正确答案）是已知的，模型要学的就是怎么从面积、房龄算出这个已知答案。

常见场景：

- 判断邮件是不是垃圾邮件（每封邮件都提前标好了"垃圾/正常"）
- 判断图片里是不是猫（每张图都标好了"是猫/不是猫"）
- 预测房价、预测明天股票涨跌（历史数据里，真实价格、真实涨跌都是已知的）

**什么时候用**：能拿到"标好答案"的数据时优先用监督学习，效果通常最好、最稳定，因为模型有一个明确的目标去逼近。缺点是标注数据往往要靠人工，费时费钱（比如标 100 万张猫的图片，得有人一张张看、一张张标）。

### 无监督学习（Unsupervised Learning）

**一句话理解**：给模型看的数据完全没有标准答案，模型自己去找数据里藏着的规律或结构。

**例子**：把顾客的消费数据（不带任何标签）丢给模型，模型自己把消费习惯相似的顾客"聚"成几类——可能聚出"高频小额消费群"、"低频大额消费群"，这些类别是模型自己发现的，没人事先告诉它该怎么分。

常见场景：

- 电商网站把用户自动分成几类消费群体（聚类），用来做精准营销
- 把一堆新闻文章按主题自动分组
- 银行监控信用卡交易，找出"不像正常消费模式"的异常交易（没人标注哪笔是欺诈，模型自己发现"不合群"的数据点）

**什么时候用**：拿不到标签、或标注成本太高时用；也常用来"探索数据"，看看里面有没有你没想到的规律。缺点是没有标准答案，很难直接判断"模型学得对不对"，聚出来的类别往往还需要人再解读含义。

### 半监督学习（Semi-Supervised Learning）

**一句话理解**：大部分数据没有标签，只有一小部分数据有标签，模型两种数据都用——先靠少量有标签数据学个大概方向，再借助大量无标签数据把细节磨准。

**例子**：训练一个模型识别"肺部 CT 图里有没有结节"。请医生标注 10 万张 CT 图非常贵、非常慢，但医院手里有 10 万张 CT 图，其中可能只有 2000 张被医生标注过。半监督学习的做法是：先用这 2000 张有标签的数据训练一个初步模型，再让模型去看剩下 9.8 万张没标签的图，模型自己给这些图打一个"猜测标签"，把猜得比较有把握的也当成新的训练数据，反复迭代——这样只标注 2% 的数据，效果也能接近标注了全部数据。

常见场景：

- 医学影像诊断（标注需要专业医生，成本极高，但原始影像数据量很大）
- 语音识别（录音很容易大量收集，但把每段录音转成文字标注很费人工）
- 海量网页/文章自动分类（网页数量太大，人工只能标一小部分）

**什么时候用**：手里有一小撮标注数据 + 一大堆没标注数据时，半监督学习是监督学习和无监督学习之间的"性价比选择"——比纯监督学习省标注成本，又比纯无监督学习效果更好（因为多少有一点"标准答案"当锚点）。

### 三者对比

| | 监督学习 | 无监督学习 | 半监督学习 |
|-|-----------|-------------|--------------|
| 数据要不要标签 | 全部数据都有标签 | 全部数据都没有标签 | 少量有标签 + 大量没标签 |
| 模型学的是什么 | 输入 → 已知答案的对应关系 | 数据里自己藏着的结构/规律 | 先靠少量答案定方向，再靠大量数据磨细节 |
| 优点 | 效果通常最好、最可控 | 不需要标注成本 | 标注成本低，效果比纯无监督好 |
| 缺点 | 标注数据贵、耗时 | 没有标准答案，效果难评估 | 需要一部分标注数据打底，流程更复杂 |
| 举例 | 垃圾邮件分类、房价预测、图像识别（猫/狗） | 用户聚类、异常检测、新闻自动分组 | 医学影像诊断、语音识别、大规模网页分类 |

> **类比**：监督学习像做"带标准答案的练习册"，每道题后面都印着答案，你对着答案学；无监督学习像拿到一大堆没分类的照片，让你自己把长得像的分成几堆，没人告诉你分得对不对；半监督学习则像老师只批改了你前 10 道题，剩下 90 道题你要靠这 10 道题总结出的规律自己去猜、自己去对。

> **和前面内容的关系**：前面讲线性回归、DNN 训练权重时用的例子（面积→价格、图片→是不是猫），全部都是"有标签数据"，所以都属于监督学习。深度学习里也有大量无监督/半监督的应用（比如自编码器、对比学习），后面会陆续展开——这里先建立"根据数据有没有标签"这个划分学习方式的大框架。

## 数据集的划分：训练集、验证集、测试集

**一句话理解**：把收集到的数据切成几份，分别用来"训练模型"、"调参数选模型"、"最终打分"——这样才能诚实地知道模型到底学得好不好。

**为什么不能一份数据从头用到尾**：类比准备考试——如果复习时用的就是考试原题、连答案都背下来了，考出 100 分，并不代表真的学会了，只能说明记住了这几道题。模型也一样：如果拿同一批数据既训练又打分，模型很可能只是"死记硬背"了这批数据（这种现象叫**过拟合，overfitting**），换一批没见过的新数据，表现可能很差。所以要把数据切开，模拟"平时练习"和"正式考试"分开的场景。

### 训练集（Training Set）

**一句话理解**：真正拿来"喂给模型学"的数据——前面讲的"调权重、调偏置"，用的就是这部分数据。

**例子**：收集了 1000 套房子的"面积、房龄 → 成交价"数据，其中 700 套划为训练集。模型反复看这 700 套数据，不断调整 w、b，让预测值越来越接近真实成交价。

### 验证集（Validation Set）

**一句话理解**：训练过程中用来"抽查、选设计"的数据——模型不会直接拿它来调权重，但会看它的表现，来决定要不要调整网络的设计（比如隐藏层数量、学习率等超参数）。

**例子**：从这 1000 套房子里再拿出 150 套作为验证集。假设先设计一个 3 层网络训练完，在验证集上误差是 20 万；换成 5 层网络训练完，误差降到 12 万——于是选择 5 层网络这个设计。验证集的作用，就是帮你在"不同的模型设计/超参数"之间做选择，而不用碰真正的测试集。

### 测试集（Test Set）

**一句话理解**：模型训练、调参全部完成之后，最后才拿出来"打分"的数据，全程不参与训练和调参，模拟模型真正上线后会遇到的新数据。

**例子**：剩下 150 套房子作为测试集。等 5 层网络的设计、超参数都定好、训练也完成之后，才第一次让模型看这 150 套房子，算出最终误差——这个误差才是模型"真实水平"的可信估计，因为这批数据模型一次都没见过、也没被用来做任何选择。

**三者的核心区别，一句话总结**：训练集用来"学"，验证集用来"选"，测试集用来"考"。测试集必须从头到尾"雪藏"，一次都不能被用来调整模型，否则这个"考试成绩"就不可信了（相当于考试前偷看了答案）。

### 常见划分比例

没有绝对标准，取决于数据总量：

| 数据总量 | 常见划分比例（训练 : 验证 : 测试） |
|-|-|
| 较小（几千条以内） | 60% : 20% : 20% |
| 较大（几十万条以上） | 98% : 1% : 1% |

> **为什么数据量大时验证/测试的比例可以变小**：验证集、测试集的作用是"评估"，只要绝对数量够多（比如几千条），就足以算出一个可靠的误差估计，不需要占很大比例——省下来的数据全部拿去训练，模型能学到的东西反而更多。

### 划分时要注意的坑

- **要先随机打乱，再划分**：如果数据是按时间或类别排好序的（比如房价数据按小区排列），直接按顺序切前 70%/中间 15%/后 15%，可能导致训练集和测试集里的房子来自完全不同的小区，测试结果会很不准。正确做法是先把数据顺序打乱，再随机切分。
- **测试集绝对不能"偷看"**：如果发现测试集上效果不好，就跑回去改模型设计、调参数，再拿测试集验证——反复这样做几次，测试集实际上变相参与了调参过程，最终分数就不再可信了。这也是验证集存在的意义：把"偷看、调整"这件事放到验证集上做，测试集才能保持干净，只在最后用一次。
- **三份数据之间不能有重复**：同一条数据不能既在训练集又在测试集里出现，否则模型相当于在"考试"时遇到了做过的原题。

> **类比**：训练集是平时刷的习题册（可以反复看、对答案、总结错题）；验证集是模拟考（可以用来判断复习方法对不对、要不要换参考书，考完还能继续调整学习计划）；测试集是最终的高考（只考一次，考完就是最终成绩，不能因为考砸了就要求重考、或者回去改答案）。

## 什么是深度学习（DL）

**一句话理解**：深度学习 = 机器学习的其中一种方法，专门用"多层神经网络"来学习规律。

**例子**：还是判断猫的图片。

- 传统机器学习做法：先由人设计好"特征提取"规则（比如提取边缘、颜色直方图），再把这些特征交给模型（比如 SVM）去分类——特征还是人想出来的。
- 深度学习做法：直接把原始像素丢进多层神经网络，网络自己一层层学会"什么是边缘 → 什么是形状 → 什么是耳朵/眼睛 → 这是猫"，特征也是自己学出来的，不需要人设计。

## 三者的关系

三者是一层套一层的**包含关系**，范围从大到小依次是：人工智能 ⊃ 机器学习 ⊃ 深度学习。

```
┌──────────────────────────────────────────────┐
│  人工智能 AI                                    │
│  让机器表现出类人智能的一切方法（规则、搜索、学习……）│
│                                                │
│   ┌────────────────────────────────────────┐  │
│   │  机器学习 ML                              │  │
│   │  从数据中自动学习规律的一类方法             │  │
│   │                                          │  │
│   │    ┌────────────────────────────────┐  │  │
│   │    │  深度学习 DL                       │  │  │
│   │    │  用多层神经网络实现的机器学习       │  │  │
│   │    └────────────────────────────────┘  │  │
│   └────────────────────────────────────────┘  │
└──────────────────────────────────────────────┘
```

对照表：

| | 人工智能 AI | 机器学习 ML | 深度学习 DL |
|-|--------------|--------------|--------------|
| 范围 | 最大，是总称 | AI 的子集 | ML 的子集 |
| 核心思路 | 让机器表现出智能行为，不限手段 | 从数据中自动总结规律 | 用多层神经网络自动学习，连"特征"都自己学 |
| 需不需要人设计规则/特征 | 可以完全靠人写规则 | 规则自动学，但特征常需要人设计 | 规则和特征都自动学 |
| 举例 | 下棋程序、语音助手、以下两者都算 | 垃圾邮件过滤、决策树、推荐系统 | 图像识别（YOLO）、语音识别、大语言模型 |

> **记忆技巧**：人工智能是"目标"（让机器变聪明），机器学习是"手段之一"（从数据里学），深度学习是"手段中最热门的一种"（用神经网络学）。三个词经常被混用，但范围是一层比一层小、一层比一层具体。

## 什么是深度神经网络（DNN）

**一句话理解**：深度神经网络 = 很多层"神经元"叠在一起组成的一个函数，隐藏层的数量多（一般 ≥ 2 层）就叫"深"。

**先看一个神经元在做什么**：一个神经元的计算很简单——把上一层传来的每个数字，各自乘上一个权重、加起来，再加一个偏置，最后过一道"激活函数"决定要不要往下传：

```
神经元输出 = 激活函数( w1×x1 + w2×x2 + ... + wn×xn + b )
```

比如输入 x1=3、x2=5，权重 w1=0.5、w2=-0.2，偏置 b=1：

```
先算加权和：0.5×3 + (-0.2)×5 + 1 = 1.5 - 1 + 1 = 1.5
再过激活函数（比如 ReLU：负数变 0，正数不变）：ReLU(1.5) = 1.5
这个神经元最终输出 1.5，传给下一层继续算
```

**神经网络就是把这样的神经元，一层一层摆很多个、连起来**：

```
输入层         隐藏层1        隐藏层2        输出层
 x1 ──┬──→ ○ ──┬──→ ○ ──┬──→
      ├──→ ○ ──┼──→ ○ ──┼──→  ○ → 输出（比如"是猫"的概率）
 x2 ──┴──→ ○ ──┴──→ ○ ──┴──→
       （每条连线都有自己的权重，每个 ○ 都有自己的偏置）
```

- **输入层**：原始数据进来的地方，比如图片的每个像素、或房子的"面积、房龄"这两个数字。
- **隐藏层**：中间层，一层层"加工"数据。层数越多、每层神经元越多，能学会的规律就越复杂。
- **输出层**：给出最终答案，比如"是猫"的概率、或预测的房价。

**"深"具体指什么**：就是隐藏层的层数多。早期的神经网络常常只有 0～1 层隐藏层，能力有限；深度学习里说的"深度神经网络"通常有几十甚至上百层隐藏层——前一层学会识别"边缘"，下一层在此基础上学"形状"，再下一层学"眼睛耳朵"……层数越深，自动学到的特征就越抽象、越高级。

> **类比**：如果一个神经元是"加权求和 + 判断"的小工人，浅层网络就像只有一个工人在干活；深度网络则是几十上百个工人排成很多道工序，前一道工序的产出是后一道的原料，越往后加工出来的东西越"精细"（像素 → 边缘 → 形状 → 五官 → 这是一张猫脸）。

| | 浅层网络 | 深度神经网络（DNN） |
|-|-----------|------------------------|
| 隐藏层数量 | 0～1 层 | 通常 ≥ 2 层，深的可以到几十~上百层 |
| 能学到的规律 | 比较简单（接近直线/简单曲线） | 复杂、抽象（图像里的物体、语义等） |
| 举例 | 线性回归、单层感知机 | YOLO、大语言模型里的网络主干 |

## 深度学习的本质：在找一个函数

**这句话基本对，但更精确的说法是**：深度学习训练的过程，是在一个由无数"权重 + 偏置"组成的巨大空间里，通过不断微调这些数字，找到一个能把输入（比如一张图片）正确映射到输出（比如"这是猫"）的函数。严格来说，"权重"不是长在神经元上的，而是长在**每一条连接**上；每个神经元自己还额外带一个"偏置"。所以更准确的说法是：**找到每一条连接的权重、和每个神经元的偏置**。

**从最简单的例子理解"找函数"这件事**：先不看神经网络，看最基础的线性回归。假设你有几组房屋"面积 → 价格"的数据，想找一条直线 `y = w·x + b` 去拟合它们：

```
面积(x)   价格(y，万元)
50        150
80        220
120       310

想找到 w、b，使得直线 y = w·x + b 尽量穿过这些点
比如最后学出 w=2.1, b=35：
y = 2.1×100 + 35 = 245（预测 100㎡ 大概卖 245 万）
```

这其实已经是"深度学习"最迷你的版本——**一个只有 1 个神经元、没有隐藏层、没有激活函数的网络**。你在"找"的东西，就是 `w` 和 `b` 这两个数字。神经网络只是把这个套路重复了成千上万次：每一层有很多神经元，每个神经元连着上一层的所有神经元，每条连接都有自己的权重，叠出一个远比一条直线复杂得多的函数。

**神经网络架构 = 函数的"形状"，权重 = 函数的"具体样子"**：一个只有一个隐藏层的迷你网络，算出的整个公式本身就是一个函数：

```
y = Sigmoid( w3 × ReLU(w1·x1 + w2·x2 + b1) + b2 )
```

- 有几层、每层几个神经元、用什么激活函数——这些是人在设计网络时就定死的，决定了这个函数"能长成什么样子"（比如是直线还是能拐很多弯的曲线），叫**超参数**，训练过程不会去调它们。
- `w1, w2, w3, b1, b2` 这些具体数字——这些才是训练要找的东西，叫**参数**。训练开始前它们是随机数，此时这个函数基本就是乱猜；随着一轮轮"算误差 → 反向传播算梯度 → 微调参数"（这部分之后会单独展开），这些数字才会慢慢收敛，函数才变成一个"输入图片、输出正确答案"的靠谱函数。

> **类比**：把网络架构想象成一台装了很多旋钮的机器（旋钮数量和接线方式在出厂时就定好了），每个旋钮就是一个权重或偏置。刚出厂时旋钮都是乱拧的，机器输出的都是垃圾；训练就是照着"损失函数"这个指南针，一点点把每个旋钮拧到正确位置，直到这台机器算出来的结果和你想要的答案对得上。

> **为什么神经网络能拟合几乎任何函数**：这背后有个理论叫**通用近似定理（Universal Approximation Theorem）**——只要网络里神经元足够多，理论上它能逼近任意复杂的连续函数。这也是为什么图像识别、语音识别这些"规律复杂到人根本写不出规则"的问题，深度学习能处理，而传统写死规则的方法处理不了。

## 线性回归：从一个特征，到多个特征

**先回顾单特征线性回归**：上一节"面积→价格"的例子，就是最基础的**线性回归**——只用一个特征 x（面积）去预测 y（价格）：

```
y = w·x + b
```

x、w、b 都只是单独一个数字。

**现实问题往往不止一个特征**：预测房价，只看面积并不够——房龄、离地铁站的距离、楼层……都会影响价格。这时就要用**多维特征线性回归**（也叫多元线性回归），公式变成：

```
y = w1·x1 + w2·x2 + ... + wn·xn + b
```

每个特征 xi 都配一个专属权重 wi，把所有"特征 × 权重"加起来，再加一个偏置 b，就是预测值。这里的"多维"指的是**输入特征的个数变多了**，跟"深度神经网络"的"深"不是一回事——它依然只有一层，没有隐藏层，是深度学习家族里最简单的成员。

**具体例子**：用两个特征——面积 x1（㎡）和房龄 x2（年）——预测房价 y（万元）：

```
面积(x1)   房龄(x2)   价格(y，万元)
100         5          300
80          20         190
150         2          460

假设训练之后学出 w1=2.8, w2=-2, b=20
预测一套新房子：面积120㎡、房龄10年
y = 2.8×120 + (-2)×10 + 20 = 336 - 20 + 20 = 336（预测约336万）
```

**权重的正负，就是特征影响方向的直觉体现**：w1=2.8 是正数，说明"面积越大，价格越高"；w2=-2 是负数，说明"房龄越大，价格越低"——权重不只是"影响力大小"，正负号本身也带着含义。

**向量化写法（接回点积）**：把所有特征拼成一个向量 x=(x1, x2, ..., xn)，把所有权重拼成一个向量 w=(w1, w2, ..., wn)，"加权求和"这件事，正好就是[图像识别的数学基础](/ai-yolo/math-basics)里讲过的**向量点积**：

```
y = w·x + b = w1x1 + w2x2 + ... + wnxn + b
```

好处是：不管有 2 个特征还是 2000 个特征，公式都能写成这一行，不用一个个列出来——这也是为什么"向量、点积"是深度学习最基本的数学工具。

**和上一节"神经元"接起来**：回头看神经元的公式——

```
神经元输出 = 激活函数( w1×x1 + w2×x2 + ... + wn×xn + b )
```

去掉最外层的激活函数，剩下的 `w1x1 + w2x2 + ... + wnxn + b` 正好就是多维特征线性回归本身。所以可以这样理解：**一个不带激活函数的神经元 = 一次多维特征线性回归**；深度神经网络就是把很多个"线性回归 + 非线性激活函数"的小单元一层层叠起来、串起来，才能学会远比一条直线（或一个平面）复杂的规律。

| | 单特征线性回归 | 多维特征线性回归 |
|-|-----------------|-------------------|
| 公式 | y = w·x + b | y = w1x1+w2x2+...+wnxn+b |
| 输入特征数量 | 1 个 | n 个（n ≥ 2） |
| 需要学的权重数量 | 1 个 w | n 个权重，每个特征配一个 |
| 几何图像 | 二维平面里的一条直线 | n 维空间里的一个"超平面" |
| 举例 | 只用"面积"预测房价 | 用"面积 + 房龄 + ..."预测房价 |

## "模型"到底是什么：架构 + 权重

**不完全对，少了一半**：光是"各个权重的组合"还不是完整的模型，完整的说法是：

```
模型 = 网络架构（骨架：几层、每层几个神经元、怎么连接、用什么激活函数）
     + 权重和偏置（每条连接、每个神经元具体的数字）
```

**为什么光有权重不够——具体例子**：假设有人扔给你这样一串数字，说"这就是模型"：

```python
weights = [0.6, 0.3, 0.1, -0.5, 0.8, -0.2]
```

你完全没法用它算出任何东西，因为你不知道：这 6 个数字要怎么分组？哪几个属于第一层、哪几个属于第二层？每层几个神经元？中间要不要过 ReLU？——**这些"怎么组织、怎么计算"的信息，就是架构**。同样的一堆权重数字，配上不同的架构，算出来的结果完全不同。反过来，同一个架构，换一套权重，输出也完全不同（比如刚初始化时是随机数、乱猜一气；训练好之后，是能准确识别猫的靠谱数字）。所以两者缺一不可，合在一起才叫"模型"。

对照表：

| | 架构（Architecture） | 权重/参数（Weights） |
|-|------------------------|--------------------------|
| 决定什么 | 层数、每层神经元个数、怎么连接、激活函数类型 | 每条连接、每个偏置的具体数值 |
| 什么时候确定 | 人设计模型时就定好，训练全程**不变** | 训练前是随机数，训练中**不断被调整** |
| 类比 | 房子的图纸/骨架 | 图纸上每根梁具体用多粗的钢材 |
| 常见叫法 | "网络结构" | "模型参数"、"参数量"（比如"70 亿参数的大模型"，数的就是这些数字的总数量） |

> **接回上一节的"旋钮机器"类比**：架构决定了这台机器**有多少个旋钮、旋钮之间怎么接线**（这在出厂时就焊死了，训练不会改变接线方式）；权重就是每个旋钮**具体拧到哪个刻度**。只说"一堆旋钮刻度"不算一台完整的机器——你还得知道这些旋钮是怎么接线的，两者合起来，才是能实际运转、算出结果的那台机器，也就是"模型"。

## 代价函数（Cost Function）：怎么衡量模型"预测得好不好"

**一句话理解**：代价函数 = 一个专门用来打分的公式，把模型当前的预测值和真实答案作对比，算出一个数字——数字越大，说明模型预测得越离谱；训练的目标，就是不断调整权重、把这个数字降到最小。

**先分清两个容易混的词**：

- **损失函数（Loss Function）**：衡量**一条**数据预测错了多少。
- **代价函数（Cost Function）**：把训练集里**所有**数据的损失加起来求平均，代表模型在整个训练集上的表现。

前面提到的"训练指南针"，指的正是这个代价函数——训练的每一步，都是在朝着让代价函数变小的方向调整权重 w 和偏置 b。

**最常见的代价函数：均方误差（MSE, Mean Squared Error）**：

```
代价 = (1/n) × Σ(预测值 - 真实值)²
```

n 是数据条数，Σ 表示"把每一条数据的(预测值-真实值)²都加起来"。取平方有两个作用：一是让"预测多了"和"预测少了"都变成正数，不会互相抵消；二是让差得越远的点，被惩罚得越重。

**具体例子**：还是用面积→价格的数据，4 套训练房子：

```
面积(x)   真实价格(y，万元)
50        150
80        220
120       310
160       380
```

假设当前模型是 `y = 2×x + 30`，先算出每套房子的预测价格、和真实价格的误差：

```
面积50:  预测 2×50+30=130,  真实150,  误差²=(130-150)²=400
面积80:  预测 2×80+30=190,  真实220,  误差²=(190-220)²=900
面积120: 预测 2×120+30=270, 真实310,  误差²=(270-310)²=1600
面积160: 预测 2×160+30=350, 真实380,  误差²=(350-380)²=900

代价 = (400+900+1600+900) / 4 = 3800/4 = 950
```

这个"950"就是当前这套 w=2、b=30 的代价函数值。换一套 w、b，代价可能变成 200，也可能变成 2000——**训练神经网络，本质上就是不断调整 w、b，让这个代价值越来越小的过程**（具体怎么调，靠的是"梯度下降 + 反向传播"，这部分后面单独展开）。

> **类比**：代价函数就像扔飞镖时"离靶心的总距离"。每换一套 w、b 就相当于换一种扔法，都能算出一个"离靶心多远"的分数，分数越小说明扔得越准。训练模型，就是根据这个分数不断调整下一次怎么扔，直到扔得又准又稳。

## 欠拟合（Underfitting）与过拟合（Overfitting）

有了代价函数这把"尺子"，就能分别量出模型在训练集、测试集上"错得有多离谱"。把这两个数字放在一起看，会遇到两种常见的"翻车"情况。

### 欠拟合（Underfitting）

**一句话理解**：模型太简单，连训练集本身都学不好——**训练集代价高，测试集代价也高**。

**例子**：还是那 4 套训练房子，这次故意用一个很粗糙的模型 `y = 1.5×x + 50`：

```
面积50:  预测 1.5×50+50=125,  真实150,  误差²=(125-150)²=625
面积80:  预测 1.5×80+50=170,  真实220,  误差²=(170-220)²=2500
面积120: 预测 1.5×120+50=230, 真实310,  误差²=(230-310)²=6400
面积160: 预测 1.5×160+50=290, 真实380,  误差²=(290-380)²=8100

训练集代价 = (625+2500+6400+8100)/4 = 17625/4 = 4406.25
```

再拿一套没见过的测试房子（面积100㎡，真实价格260万）验证：预测 = 1.5×100+50=200，误差²=(200-260)²=3600——**同样很离谱**。训练集代价 4406.25、测试代价也很高，两边都学不好，说明模型本身"脑容量不够"，压根没抓住"面积→价格"这条规律，直线的斜率和截距都偏得太多。

**常见原因**：模型太简单（该用曲线的地方硬用直线）、给的特征太少（比如预测房价只给了面积，没给房龄、地段）、训练轮数太少（还没学到位就停了）。

**怎么解决**：换更复杂一点的模型（加隐藏层、加神经元）、加更多有用的特征、多训练几轮。

### 过拟合（Overfitting）

**一句话理解**：模型太复杂，把训练集"死记硬背"了下来，连数据里的偶然波动、噪声都背了下来——**训练集代价极低（甚至接近 0），测试集代价却很高**。

**例子**：同样 4 套训练房子，这次用一条非常"扭曲"的高次曲线，硬是让它精确穿过每一个训练点：

```
训练集代价 ≈ 0（曲线被强行调整到刚好穿过这 4 个点，误差几乎为 0）
```

看起来训练集上表现"完美"，但拿去测试面积100㎡、真实价格260万的那套房子：因为这条曲线为了穿过每个训练点，在点与点之间被迫剧烈上下摆动，预测值可能变成 500 万——误差²=(500-260)²=57600，**离谱到爆炸**。这就是过拟合：模型根本没学到"面积越大、价格越高"这条真规律，只是硬背下了这 4 个点的坐标，换个新数字就露馅了。

**常见原因**：模型太复杂（层数/神经元太多，能力过剩）、训练数据太少、训练轮数太多（后期在死记硬背训练集里的细节和噪声）。

**怎么解决**：简化模型、增加训练数据、提前停止训练（early stopping）、用"正则化"惩罚过大的权重（这些方法后面会展开）。

### 三者放在一起对比

| | 欠拟合 Underfitting | 刚好合适 Good Fit | 过拟合 Overfitting |
|-|----------------------|----------------------|----------------------|
| 训练集代价 | 高 | 较低 | 极低（接近 0） |
| 测试集代价 | 高 | 较低 | 高 |
| 模型学到了什么 | 规律都没抓住 | 抓住了真实规律 | 背下了训练数据，连噪声都背了 |
| 本质原因 | 模型太简单 / 特征太少 / 训练不够 | 复杂度刚好匹配数据规律 | 模型太复杂 / 数据太少 / 训练过头 |
| 上面例子里的代价 | 训练4406.25，测试误差²=3600 | 训练950，测试误差²=900 | 训练≈0，测试误差²=57600 |

**这也是为什么"数据集要分训练集、验证集、测试集"这件事这么重要**（回顾前面「数据集的划分」这一节）：如果只看训练集代价，欠拟合和过拟合都可能被掩盖——过拟合的训练集代价看起来"完美无缺"，只有测试集才能戳穿真相。所以调参、选模型时一定要看**验证集/测试集**上的代价，而不是只盯着训练集代价。

> **类比**：欠拟合像是复习太少，练习册、正式考试都考砸，因为压根没学会方法；过拟合像是死记硬背了练习册上每道题的答案（练习册能考满分），但考试换了几个数字、改了改问法就完全不会了；刚好合适，则是真正理解了解题方法，练习册和考试都能考出不错的分数——这正好呼应前面训练集/验证集/测试集里"练习册 vs 正式考试"的类比。
