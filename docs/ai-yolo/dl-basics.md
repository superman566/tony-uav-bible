# 深度学习基础

## 什么是人工智能（AI）

**一句话理解**：人工智能 = 让机器表现出"像人一样聪明"的行为，是一个最大的概念范畴，怎么实现都算。

只要一个程序能完成原本需要人类智能才能做的事——下棋、认脸、听懂话、开车——不管它内部是怎么实现的，都可以叫人工智能。早期很多 AI 甚至完全不需要"学习"，纯靠人写规则：

**例子**：一个简单的下棋程序，人可以直接写死规则——"如果对方吃我的车，我就把它的皇后将死"。这种靠 if-else 规则堆出来的程序，也是人工智能的一种，只是比较"笨"，规则之外的情况就不会处理了。

## 什么是机器学习（ML）

**一句话理解**：机器学习 = 实现人工智能的其中一种方法，让机器自己从数据里"总结"规律，而不是人把规则写死。

**例子**：判断一封邮件是不是垃圾邮件。

- 传统 AI 做法：人自己想规则——"包含‘中奖’‘免费’‘点击领取’就是垃圾邮件"，写成 if-else。
- 机器学习做法：给模型看 10 万封已经标好"垃圾/正常"的邮件，模型自己统计出"哪些词、哪些特征更容易出现在垃圾邮件里"，总结出一套规则。

机器学习的核心变化是：**规则不再是人写的，而是从数据里"学"出来的**。常见的机器学习方法有决策树、SVM（支持向量机）、线性回归等，深度学习也是机器学习的一种。

## 什么是深度学习（DL）

**一句话理解**：深度学习 = 机器学习的其中一种方法，专门用"多层神经网络"来学习规律。

**例子**：还是判断猫的图片。

- 传统机器学习做法：先由人设计好"特征提取"规则（比如提取边缘、颜色直方图），再把这些特征交给模型（比如 SVM）去分类——特征还是人想出来的。
- 深度学习做法：直接把原始像素丢进多层神经网络，网络自己一层层学会"什么是边缘 → 什么是形状 → 什么是耳朵/眼睛 → 这是猫"，特征也是自己学出来的，不需要人设计。

## 三者的关系

三者是一层套一层的**包含关系**，范围从大到小依次是：人工智能 ⊃ 机器学习 ⊃ 深度学习。

```
┌──────────────────────────────────────────────┐
│  人工智能 AI                                    │
│  让机器表现出类人智能的一切方法（规则、搜索、学习……）│
│                                                │
│   ┌────────────────────────────────────────┐  │
│   │  机器学习 ML                              │  │
│   │  从数据中自动学习规律的一类方法             │  │
│   │                                          │  │
│   │    ┌────────────────────────────────┐  │  │
│   │    │  深度学习 DL                       │  │  │
│   │    │  用多层神经网络实现的机器学习       │  │  │
│   │    └────────────────────────────────┘  │  │
│   └────────────────────────────────────────┘  │
└──────────────────────────────────────────────┘
```

对照表：

| | 人工智能 AI | 机器学习 ML | 深度学习 DL |
|-|--------------|--------------|--------------|
| 范围 | 最大，是总称 | AI 的子集 | ML 的子集 |
| 核心思路 | 让机器表现出智能行为，不限手段 | 从数据中自动总结规律 | 用多层神经网络自动学习，连"特征"都自己学 |
| 需不需要人设计规则/特征 | 可以完全靠人写规则 | 规则自动学，但特征常需要人设计 | 规则和特征都自动学 |
| 举例 | 下棋程序、语音助手、以下两者都算 | 垃圾邮件过滤、决策树、推荐系统 | 图像识别（YOLO）、语音识别、大语言模型 |

> **记忆技巧**：人工智能是"目标"（让机器变聪明），机器学习是"手段之一"（从数据里学），深度学习是"手段中最热门的一种"（用神经网络学）。三个词经常被混用，但范围是一层比一层小、一层比一层具体。

## 深度学习的本质：在找一个函数

**这句话基本对，但更精确的说法是**：深度学习训练的过程，是在一个由无数"权重 + 偏置"组成的巨大空间里，通过不断微调这些数字，找到一个能把输入（比如一张图片）正确映射到输出（比如"这是猫"）的函数。严格来说，"权重"不是长在神经元上的，而是长在**每一条连接**上；每个神经元自己还额外带一个"偏置"。所以更准确的说法是：**找到每一条连接的权重、和每个神经元的偏置**。

**从最简单的例子理解"找函数"这件事**：先不看神经网络，看最基础的线性回归。假设你有几组房屋"面积 → 价格"的数据，想找一条直线 `y = w·x + b` 去拟合它们：

```
面积(x)   价格(y，万元)
50        150
80        220
120       310

想找到 w、b，使得直线 y = w·x + b 尽量穿过这些点
比如最后学出 w=2.1, b=35：
y = 2.1×100 + 35 = 245（预测 100㎡ 大概卖 245 万）
```

这其实已经是"深度学习"最迷你的版本——**一个只有 1 个神经元、没有隐藏层、没有激活函数的网络**。你在"找"的东西，就是 `w` 和 `b` 这两个数字。神经网络只是把这个套路重复了成千上万次：每一层有很多神经元，每个神经元连着上一层的所有神经元，每条连接都有自己的权重，叠出一个远比一条直线复杂得多的函数。

**神经网络架构 = 函数的"形状"，权重 = 函数的"具体样子"**：一个只有一个隐藏层的迷你网络，算出的整个公式本身就是一个函数：

```
y = Sigmoid( w3 × ReLU(w1·x1 + w2·x2 + b1) + b2 )
```

- 有几层、每层几个神经元、用什么激活函数——这些是人在设计网络时就定死的，决定了这个函数"能长成什么样子"（比如是直线还是能拐很多弯的曲线），叫**超参数**，训练过程不会去调它们。
- `w1, w2, w3, b1, b2` 这些具体数字——这些才是训练要找的东西，叫**参数**。训练开始前它们是随机数，此时这个函数基本就是乱猜；随着一轮轮"算误差 → 反向传播算梯度 → 微调参数"（这部分之后会单独展开），这些数字才会慢慢收敛，函数才变成一个"输入图片、输出正确答案"的靠谱函数。

> **类比**：把网络架构想象成一台装了很多旋钮的机器（旋钮数量和接线方式在出厂时就定好了），每个旋钮就是一个权重或偏置。刚出厂时旋钮都是乱拧的，机器输出的都是垃圾；训练就是照着"损失函数"这个指南针，一点点把每个旋钮拧到正确位置，直到这台机器算出来的结果和你想要的答案对得上。

> **为什么神经网络能拟合几乎任何函数**：这背后有个理论叫**通用近似定理（Universal Approximation Theorem）**——只要网络里神经元足够多，理论上它能逼近任意复杂的连续函数。这也是为什么图像识别、语音识别这些"规律复杂到人根本写不出规则"的问题，深度学习能处理，而传统写死规则的方法处理不了。

## "模型"到底是什么：架构 + 权重

**不完全对，少了一半**：光是"各个权重的组合"还不是完整的模型，完整的说法是：

```
模型 = 网络架构（骨架：几层、每层几个神经元、怎么连接、用什么激活函数）
     + 权重和偏置（每条连接、每个神经元具体的数字）
```

**为什么光有权重不够——具体例子**：假设有人扔给你这样一串数字，说"这就是模型"：

```python
weights = [0.6, 0.3, 0.1, -0.5, 0.8, -0.2]
```

你完全没法用它算出任何东西，因为你不知道：这 6 个数字要怎么分组？哪几个属于第一层、哪几个属于第二层？每层几个神经元？中间要不要过 ReLU？——**这些"怎么组织、怎么计算"的信息，就是架构**。同样的一堆权重数字，配上不同的架构，算出来的结果完全不同。反过来，同一个架构，换一套权重，输出也完全不同（比如刚初始化时是随机数、乱猜一气；训练好之后，是能准确识别猫的靠谱数字）。所以两者缺一不可，合在一起才叫"模型"。

对照表：

| | 架构（Architecture） | 权重/参数（Weights） |
|-|------------------------|--------------------------|
| 决定什么 | 层数、每层神经元个数、怎么连接、激活函数类型 | 每条连接、每个偏置的具体数值 |
| 什么时候确定 | 人设计模型时就定好，训练全程**不变** | 训练前是随机数，训练中**不断被调整** |
| 类比 | 房子的图纸/骨架 | 图纸上每根梁具体用多粗的钢材 |
| 常见叫法 | "网络结构" | "模型参数"、"参数量"（比如"70 亿参数的大模型"，数的就是这些数字的总数量） |

> **接回上一节的"旋钮机器"类比**：架构决定了这台机器**有多少个旋钮、旋钮之间怎么接线**（这在出厂时就焊死了，训练不会改变接线方式）；权重就是每个旋钮**具体拧到哪个刻度**。只说"一堆旋钮刻度"不算一台完整的机器——你还得知道这些旋钮是怎么接线的，两者合起来，才是能实际运转、算出结果的那台机器，也就是"模型"。
