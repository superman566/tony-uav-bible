# 经典计算机视觉核心技术与算法

## 传统计算机视觉处理流程

传统方法的核心思路：**所有规则都由人工设计**，告诉计算机"怎么找特征"，而不是让它自己学。

```
原始图像
   ↓
① 预处理       — 灰度化、增强对比度、去畸变
   ↓
② 特征提取     — 找边缘、角点、纹理
   ↓
③ 分割         — 用阈值把目标和背景分开
   ↓
④ 形态学处理   — 膨胀/腐蚀，去噪点、填空洞
   ↓
⑤ 目标检测与识别 — 特征描述子 + 分类器（如 HOG + SVM）
```

### ① 预处理

让图像更"干净"，减少后续步骤的干扰。

- **灰度化**：去掉颜色通道，只保留亮度，减少计算量
- **增强对比度**：让暗处更暗、亮处更亮，特征更明显
- **去畸变 / 透视校正**：修正摄像头引起的变形

### ② 特征提取

找出图像里"有意义的地方"，是区分不同物体的关键。

| 特征类型 | 含义 | 代表算法 |
|----------|------|----------|
| 边缘 | 物体轮廓，亮度突变处 | Canny、Sobel |
| 角点 | 两条边交汇的拐角 | Harris、SIFT、ORB |
| 纹理 | 区域内的重复图案/材质感 | HOG、LBP |

### ③ 分割

用一个亮度门槛（阈值）把图像分成两部分：目标区域 和 背景，分别处理。

- **固定阈值**：手动指定门槛值
- **Otsu 自动阈值**：算法自动找最优门槛
- **自适应阈值**：图像不同区域用不同门槛（适合光照不均）

### ④ 形态学处理

对分割后的二值图像做"结构优化"，消除瑕疵。

| 操作 | 效果 |
|------|------|
| 膨胀（Dilate） | 把白色区域向外扩张，填补小空洞 |
| 腐蚀（Erode） | 把白色区域向内收缩，去除小噪点 |
| 开运算（先腐蚀后膨胀） | 去除小噪点，保持整体形状 |
| 闭运算（先膨胀后腐蚀） | 填补空洞，连接断裂区域 |

### ⑤ 目标检测与识别

把前面提取的特征输入分类器，判断图中有什么物体、在哪里。

- **特征描述子**：HOG（把梯度方向统计成向量）、SIFT（尺度不变特征）
- **分类器**：SVM（支持向量机）、AdaBoost（Haar 级联人脸检测）
- **检测框筛选**：NMS（非极大值抑制）去掉重叠的多余框

> **传统 vs 深度学习**：传统方法每一步特征都靠人工设计，泛化能力弱，但速度快、可解释性强。深度学习让网络自己学特征，精度更高但需要大量数据。

## 经典算法

### 二值化（Binarization）

把灰度图变成只有黑（0）和白（255）两种值的图像，方便后续轮廓检测、形态学处理等操作。

**核心思路**：设一个亮度门槛（阈值），高于门槛变白，低于门槛变黑。

```
像素亮度 > 阈值  →  255（白）
像素亮度 ≤ 阈值  →  0（黑）
```

**三种常用方式：**

| 方式 | 适用场景 | 说明 |
|------|----------|------|
| 固定阈值 | 光照稳定 | 手动指定一个值，简单直接 |
| Otsu 自动阈值 | 光照均匀 | 算法自动找让前景/背景分离最好的阈值 |
| 自适应阈值 | 光照不均 | 对图像不同区域单独计算阈值 |

**效果对比：**

| 灰度原图 | Otsu 二值化结果 |
|:--------:|:--------------:|
| ![灰度原图](/images/binarization-original.png) | ![二值化结果](/images/binarization-otsu.png) |

```python
import cv2

gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)

# 固定阈值
_, binary = cv2.threshold(gray, 127, 255, cv2.THRESH_BINARY)

# Otsu 自动阈值
_, otsu = cv2.threshold(gray, 0, 255, cv2.THRESH_BINARY + cv2.THRESH_OTSU)

# 自适应阈值
adaptive = cv2.adaptiveThreshold(gray, 255,
                                  cv2.ADAPTIVE_THRESH_GAUSSIAN_C,
                                  cv2.THRESH_BINARY, 11, 2)
```

### 模糊（Blur）

把每个像素替换成它周围像素的某种"综合值"，让图像变平滑。

**在计算机视觉中的作用：**
- **去噪**：消除拍摄时产生的随机噪点，避免干扰后续处理
- **边缘检测前的预处理**：Canny 第一步就是高斯模糊，防止噪点被误识别为边缘
- **降低计算量**：细节减少后，特征提取更稳定更快

---

**均值模糊** — 取周围所有像素的平均值

```
原始 3×3 区域（中间 200 是噪点）：
10  10  10
10 200  10
10  10  10

(10×8 + 200) ÷ 9 = 31

200 → 31   噪点被压下去了，但没完全消失
```

---

**中值模糊** — 把周围所有值排序，取最中间那个

```
排序：10, 10, 10, 10, [10], 10, 10, 10, 200
                       ↑ 中间值

200 → 10   噪点完全消失，去噪最强
```

---

**高斯模糊** — 加权平均，离中心越近的邻居权重越高

```
权重矩阵（中心权重最大）：
1  2  1
2  4  2
1  2  1
÷ 16

(10×1+10×2+10×1+10×2+200×4+10×2+10×1+10×2+10×1) ÷ 16 = 58

200 → 58   去噪效果介于均值和中值之间，但边缘保留最自然
```

---

**三种方式对比：**

| 方式 | 原理 | 去噪强度 | 边缘保留 | 适用场景 |
|------|------|----------|----------|----------|
| 均值模糊 | 取平均值 | 中 | 差 | 快速粗糙去噪 |
| 中值模糊 | 取中间值 | 最强 | 较好 | 椒盐噪点 |
| 高斯模糊 | 加权平均 | 中 | 最好 | 最常用，效果自然 |

```python
import cv2

# 均值模糊
blur = cv2.blur(img, (5, 5))

# 高斯模糊
gaussian = cv2.GaussianBlur(img, (5, 5), 0)

# 中值模糊
median = cv2.medianBlur(img, 5)
```

## 术语表

### RGB

**红(R) + 绿(G) + 蓝(B)** 三种颜色混合表示一个像素的颜色。

> 就像调色板，三种颜料按比例混合 = 任意颜色

每个通道值范围 0~255：
- `(255, 0, 0)` = 纯红
- `(0, 255, 0)` = 纯绿
- `(255, 255, 255)` = 白
- `(0, 0, 0)` = 黑

显示器、摄像头默认用 RGB（OpenCV 读进来是 BGR，顺序反了）。

---

### YUV

把图像信息拆成两部分：**亮度 + 颜色**。

| 通道 | 含义 | 类比 |
|------|------|------|
| Y | 亮度（明暗） | 黑白电视的画面 |
| U、V | 色差（颜色偏向） | 在黑白基础上"染色" |

> 人眼对亮暗比对颜色敏感得多，所以可以保留完整 Y，压缩 U/V，视频文件因此变小。

**Y 通道单独拿出来就是灰度图**，这就是为什么 YUV → Gray 只需要取 Y 通道。

---

### RGB vs YUV

| | RGB | YUV |
|-|-----|-----|
| 用途 | 显示、拍摄 | 视频编码、传输 |
| 存颜色方式 | 三原色混合 | 亮度 + 色差分离 |
| 转灰度 | 需加权计算 | 直接取 Y |
| 文件大小 | 较大 | 压缩后更小 |
